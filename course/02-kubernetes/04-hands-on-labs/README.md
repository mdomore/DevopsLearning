# Hands-on Labs (Create + Verify)

All examples run in namespace `kube-lab`.

Each lab follows the same structure:

1. **Explanation** — what problem we solve and why
2. **Key fields** — what important YAML lines mean (before you paste)
3. **Implementation** — commands to run in your **local terminal**
4. **Verification** — how to know it worked
5. **Break & repair** — one common mistake and how to fix it
6. **You should now be able to…** — checklist before moving on
